# Сопроводительный документ
Итоговый проект

Тема: Прогнозирование вероятности приобретения туристических услуг клиентом

Источник данных: датасет раздела «Путешествия» Т-Банк
## 1. Постановка задачи
### Бизнес-задача
Персонализировать первый оффер для клиентов банка, которые ранее не пользовались разделом «Путешествия».

Необходимо определить, что с большей вероятностью заинтересует клиента:

•	авиабилет (AIR)

•	бронирование отеля (HOT)
### Цель работы
Разработать ML-модель бинарной классификации, предсказывающую вероятность выбора продукта (AIR = 1, HOT = 0) на основе демографических, финансовых и поведенческих характеристик клиента, известных до момента совершения первой покупки
### Ожидаемый результат
•	Модель с устойчивым ROC-AUC > 0.7

•	Интерпретируемые факторы, влияющие на выбор продукта

•	Практические рекомендации для маркетинга
## 2. Описание этапов работы
### Загрузка и первичная обработка данных
Данные загружены из CSV-файла.

Проведена проверка на:

•	дубликаты

•	корректность типов

•	период наблюдения

Временной диапазон:

1 января – 7 ноября 2024 года (311 дней)

### Очистка и предотвращение утечки данных (Data Leakage)
Ключевой принцип:

в модель включаются только признаки, известные до момента первого заказа.

Удалены:

•	ID (account_rk, client_rk, order_rk)

•	признаки факта совершённого заказа

•	параметры конкретного бронирования

•	коммуникационная история после заказа

•	признаки с >90% пропусков

Это предотвращает переобучение и делает модель применимой к новым клиентам.

### Формирование целевой переменной

На основе order_type_cd:

•	AIR - 1

•	HOT - 0

Распределение:

•	AIR — 70%

•	HOT — 30%

Имеется умеренный дисбаланс классов.

### Работа с пропусками и выбросами

Подход:

•	Категориальные признаки - новая категория "Unknown"

Обоснование: введенение новой категории "Unknown" сохраняет честное распределение признаков

•	Доход обработка выбросов логарифмирование пропуски заполнены медианой расчитаной на исходных значениях

•	Баланс на начало мясяца обработка выбросов и пропусков выделены:

o	флаг отрицательного значения

o	модуль баланса

o	логарифм

o	знак

o пропуски заполнены медианой расчитанной на исходных значениях

Обоснование:

логарифмирование уменьшает влияние выбросов и снижает асимметрию распределений.

### Анализ числовых признаков признаков

Доход

•	Среднее значительно выше медианы

•	Сильная правосторонняя асимметрия

•	Тест Манна–Уитни показал отсутствие статистически значимых различий между AIR и HOT (p = 0.26)

Доход не является ключевым фактором выбора.

### Выбор модели

Выбран CatBoost, так как:

•	большое количество категориальных признаков

•	высокая кардинальность (география, место рождения)

•	встроенная обработка категорий

•	устойчивость к переобучению

Метрики:

•	ROC-AUC

•	PR-AUC

•	F1 для класса HOT

Accuracy не используется из-за дисбаланса.
## 3. Результаты моделирования
Модель с признаком birth_place

ROC-AUC = 0.784

PR-AUC = 0.899

F1 = 0.84

Анализ влияющих факторов показал:

•	birth_place — 29.7%

•	loyalty_program_type_nm — 8.9%

•	lvn_state_nm — 7.1%

•	age — 6.7%

Признак birth_place имел чрезмерно высокую важность.

Проверка гипотезы о переобучении

Модель обучена без birth_place.

Результат:

ROC-AUC = 0.736

PR-AUC = 0.858

F1 = 0.83

Падение метрики (~0.05) указывает, что признак действительно информативен, но возможен эффект высокой кардинальности.

 Ключевой результат

## Практическое применение

Модель может использоваться для:

•	персонализации первого оффера

•	A/B тестирования маркетинговых гипотез

•	приоритизации коммуникаций

Например:

•	Молодым клиентам из определённых регионов предлагать авиабилеты

•	Клиентам с определённой программой лояльности — отели
